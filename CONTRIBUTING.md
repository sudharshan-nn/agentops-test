# Contributing to AgentOps

Thank you for considering contributing to the AgentOps project! We welcome contributions from everyone.

## Code of Conduct

This project adheres to the Contributor Covenant code of conduct. By participating, you are expected to uphold this code. Please report unacceptable behavior to project maintainers.

## How to Contribute

1. **Fork the Repository**: Use the fork button to create your own copy of the repository.
2. **Clone the Repository**: Clone your fork to your local machine with `git clone <repository-url>`.
3. **Create a Branch**: Make a branch for your changes with `git checkout -b feature/YourFeatureName`.
4. **Make Changes**: Implement your changes to the codebase.
5. **Test Your Changes**: Ensure that any changes you've made are properly tested.
6. **Push to Your Fork**: Push your changes back to your forked repository.
7. **Submit a Pull Request (PR)**: Open a PR to merge your changes into the upstream repository.

## Pull Request Guidelines

- Ensure your PR is up-to-date with the latest changes in the main branch.
- Provide a clear description of your changes and the problem they solve.
- If your PR fixes a bug, include the issue number.
- Write clear, descriptive commit messages.

## Code Style

We follow the JavaScript Standard Style. Please ensure your code complies by running the linter before submitting your PR.

Thank you for contributing to the AgentOps project!
